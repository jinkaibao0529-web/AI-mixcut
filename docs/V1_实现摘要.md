# V1 实现摘要

本文档汇总电商短视频批量生成 V1 的代码级实现范围。

## 前端

主要文件：

- `desktop/src/App.tsx`
- `desktop/src/styles.css`

新增或扩展的工作区：

- 素材库
  - 支持导入成片视频、产品镜头、权益镜头、真人口播、BGM、配音、字幕预设
  - 产品镜头视觉打标与人工校对
  - 成片素材一键创建去 BGM 任务
  - 统一 AI Worker 任务列表
- 文案中心
  - 手动文案裂变
  - 抖音来源结构改写
  - 按来源和关键词筛选文案
  - 句子级文案编辑
  - TTS 配音生成
  - 按文案语义生成时间线
- 配音/BGM
  - TTS 生成
  - BGM 和配音试听
  - 音频响度分析与 LUFS 展示
- 字幕模板
  - 基于 metadata 的 ASS 字幕样式编辑
  - 支持字体、字号、颜色、描边、阴影、位置、边距配置
- 素材混剪时间线
  - 原音频策略控制
  - 配音/BGM/字幕预设选择
  - 统一响度控制
  - 音频策略推荐
  - 片段选择理由持久化
  - 替换候选评分与理由展示
- 素材库内的片段管理
  - 卖点 tag 和画面 tag
  - 画面/卖点描述编辑
  - 按语义、卖点、画面 tag 导出文件夹

## 媒体处理

基于 FFmpeg 实现：

- 时间线导出和原音频策略处理
- 配音替换和配音叠加
- BGM 叠加
- 通过 `loudnorm` 做导出统一响度
- 通过 `loudnorm` 做音频响度分析
- ASS 字幕生成和烧录
- 从 `.ass` 提取字幕样式
- 从 `.json` 解析字幕样式
- 通过 metadata 覆盖字幕样式

## AI Provider

通过 Provider 接入的能力：

- 产品视觉理解和卖点 tag 生成
- 文案生成、拆解、改写和裂变
- 可配置 TTS Provider

V1 占位策略：

- 未配置 TTS Provider 时，可生成静音占位音频，保证“文案到时间线”的流程可以先跑通。

## 可选 AI Worker 边界

桌面端和主后端保持轻量。重模型能力统一通过可选 `ai_tasks` 表示。

当前已预留的 Worker 任务：

- `remove_bgm`

后续可接入的 Worker 能力：

- Demucs/UVR 人声和 BGM 分离
- ForcedAligner 字级字幕对齐
- 音色克隆
- 数字人生成

## 验证方式

代码级最低验证：

```bash
./scripts/v1_smoke.sh
```

发布级验收：

- 按 `docs/V1_验收清单.md` 执行
- 使用真实成片视频、产品镜头、BGM、配音素材和字幕预设
- 手动听检导出音频
- 在手机观看导出视频并检查烧录字幕可读性
